Principal's Mental Model: Heaps & Tries

When to Apply

  • "Top K" Elements:
    Use a Heap. Sorting takes $O(N \log N)$, but Heap takes $O(N \log K)$.
  • Dynamic Data (Streaming):
    If data is coming in one by one and you need the "Current Max/Min" instantly, Arrays fail. Heaps excel.
  • Prefix Search / XOR Problems:
    Use a Trie. If the problem involves matching *starting* characters or bits, HashMap is useless. You need a tree of prefixes.

The "10x" Safety Check

  • The Inverse Logic (Heap):
    Find K-th Largest? Use MIN-Heap.
    Find K-th Smallest? Use MAX-Heap.
    Why? Because the root kicks out the "weakest" element, keeping the "Top K" inside.
  • The Crash (Heap):
    Don't push `(value, object)` into a heap. If values are equal, Python compares objects and crashes. Push `(value, unique_id, object)`.
  • Python's Default:
    Python only has Min-Heap. For Max-Heap, push negative values (`-x`).
Kth Largest in Stream
MUST DO Min-Heap Pattern
O(N log K) Time
O(K) Space
Visual: The VIP Room (Capacity K)
Heap ek VIP room hai jisme sirf K log aa sakte hain.
Darwaaze pe "Bouncer" khada hai (Heap ka Root/Min).
Agar naya number Bouncer se bada hai, toh Bouncer ko laat maarke nikaal do, aur naye ko andar le lo.
üí° The Crux Inverse Logic Rule:
Agar Kth Largest chahiye -> Toh Min-Heap use karo.
1. Heap ka size humesha `K` rakho.
2. Jaise hi size `K` se bada ho, `heapq.heappop()` kar do (sabse chota element nikaal do).
3. Jo element heap ke top pe bachega, wahi Kth Largest hai.
‚ö†Ô∏è The Trap Sorting vs Heap:
Interview mein galti se bhi `array.sort()` (O(N log N)) mat karna har `add()` operation pe.
Doosri galti: Saare `N` elements ko Heap mein mat daalna (Space O(N) ho jayega). Hamein sirf `K` elements store karne hain (Space O(K)).
Python Solution (heapq) 
    import heapq
    
    class KthLargest:
        def __init__(self, k: int, nums: List[int]):
            self.k = k
            self.heap = []
            
            for n in nums:
                self.add(n)
    
        def add(self, val: int) -> int:
            # Case 1: Agar VIP room full nahi hai, bas add karo
            if len(self.heap) < self.k:
                heapq.heappush(self.heap, val)
                
            # Case 2: Room full hai. Check karo kya naya banda Bouncer se ameer hai?
            elif val > self.heap[0]:
                # Push new val, Pop old min (Bouncer)
                # heapreplace is faster than heappushpop
                heapq.heapreplace(self.heap, val) 
                
            # Heap ka root (top) humesha Kth Largest element hoga
            return self.heap[0]
Merge K Sorted Lists
MUST DO Heap + Linked List
O(N log K) Time
O(K) Space

Visual: The Tournament
Heap mein humesha sirf `K` nodes honge (Har list ka current Head).
1. Heap se Minimum nikalo (Winner).
2. Winner ko result list mein lagao.
3. Winner ki list se next node ko Heap mein daal do.
üí° The Crux Algorithm Loop:
1. Init: Har list ka head Heap mein daalo.
2. Loop: Jab tak Heap empty na ho:
   -> `pop()` smallest node.
   -> `curr.next = node`.
   -> Agar `node.next` exist karta hai, toh usko `push()` karo.
‚ö†Ô∏è The Trap The Crash (Object Comparison):
Python ka `heapq` tuples ko compare karta hai: `(val, node)`.
Agar do nodes ki `val` same hai (e.g., dono 5 hain), toh Python `node1 < node2` compare karega.
Kyunki `ListNode` class mein `<` operator define nahi hota, code Crash kar jayega.
Fix: Use a tie-breaker: `(val, unique_index, node)`.
Python Solution (Tuple Tie-Breaker) 
    import heapq
    
    def mergeKLists(lists):
        # Heap stores: (value, unique_index, node)
        pq = []
        
        # 1. Initialize: Push head of every list
        for i, l in enumerate(lists):
            if l:
                # 'i' is unique, so comparison never reaches the 'node' object
                heapq.heappush(pq, (l.val, i, l))
                
        dummy = ListNode(0)
        curr = dummy
        
        # 2. Process
        while pq:
            val, i, node = heapq.heappop(pq)
            
            curr.next = node
            curr = curr.next
            
            if node.next:
                # Important: Use same index 'i' or a new one, 
                # as long as the tuple structure is consistent.
                # Using 'i' again is fine since the previous node from 'i' is gone.
                heapq.heappush(pq, (node.next.val, i, node.next))
                
        return dummy.next
Trie (Prefix Tree)
MUST DO System Design Core
O(L) Time
O(N*L) Space

Visual: The Autocomplete Tree
Root khali hota hai.
"APPLE" -> Root se 'A', phir 'P', phir 'P', 'L', 'E'.
"APP" -> Root se 'A', phir 'P', phir 'P'.
Iska matlab "APP" aur "APPLE" pehle 3 steps (nodes) share karte hain (Space Saving!).
üí° The Crux The Node Structure:
TrieNode ka class bahut simple hota hai:
1. children = {}: Ek HashMap jo Character map karta hai next Node pe.
2. is_end = False: Ye batata hai ki kya yahan koi word khatam ho raha hai?
‚ö†Ô∏è The Trap Prefix vs Word:
Agar Dictionary mein sirf "APPLE" hai, aur hum "APP" search karein:
search("APP") should return False (kyunki "APP" pura word nahi hai).
startsWith("APP") should return True.
Ye distinction sirf is_end flag se handle hoti hai.
Python Solution (Class Design) 
    class TrieNode:
        def __init__(self):
            self.children = {}
            self.is_end = False
    
    class Trie:
        def __init__(self):
            self.root = TrieNode()
    
        def insert(self, word: str) -> None:
            curr = self.root
            for char in word:
                if char not in curr.children:
                    curr.children[char] = TrieNode()
                curr = curr.children[char]
            # Mark the end of the word
            curr.is_end = True
    
        def search(self, word: str) -> bool:
            curr = self.root
            for char in word:
                if char not in curr.children:
                    return False
                curr = curr.children[char]
            # Return True only if it's marked as a complete word
            return curr.is_end
    
        def startsWith(self, prefix: str) -> bool:
            curr = self.root
            for char in prefix:
                if char not in curr.children:
                    return False
                curr = curr.children[char]
            return True
Max XOR of Two Numbers
GOOD Trie + Bitwise
O(32*N) Time
O(32*N) Space

Visual: The Greedy Path
Imagine Trie mein '0' Left child hai aur '1' Right child.
Agar mere paas bit '1' hai, main Trie mein '0' dhoondunga (taaki 1^0 = 1 mile).
Agar '0' mil gaya -> Success! (Add 2^k to result).
Agar '0' nahi mila -> Majboori (Go to '1', result remains same).
üí° The Crux Strategy:
1. Insert: Saare numbers ko 32-bit binary strings (padding ke saath) bana ke Trie mein daal do.
2. Query: Phir se har number ke liye Trie traverse karo.
3. Greedy Choice: Har bit ke liye, check karo "Kya Opposite Bit exist karta hai?"
  -> Yes: Udhar jao, aur answer mein `1 << i` add karo.
  -> No: Same bit wali side jao (Answer mein 0 add hoga).
‚ö†Ô∏è The Trap Fixed Length:
Numbers ko hamesha fixed length (e.g., 32 bits) mein convert karna zaroori hai.
Example: `5` should be `000...0101`, not just `101`.
Kyunki MSB (Leftmost bits) compare karna sabse zaroori hai, aur bina padding ke alignment kharab ho jayegi.
Python Solution (Greedy Bit Logic) 
    class TrieNode:
        def __init__(self):
            self.children = {}
    
    class Solution:
        def findMaximumXOR(self, nums: List[int]) -> int:
            root = TrieNode()
            
            # 1. Build Trie (Insert Binary Strings)
            for num in nums:
                node = root
                # Convert to 32-bit binary: 5 -> '00...0101'
                # zfill(32) adds leading zeros
                bin_str = bin(num)[2:].zfill(32) 
                for bit in bin_str:
                    if bit not in node.children:
                        node.children[bit] = TrieNode()
                    node = node.children[bit]
            
            max_xor = 0
            
            # 2. Query Logic (Find Best Match for each Num)
            for num in nums:
                node = root
                bin_str = bin(num)[2:].zfill(32)
                curr_xor = 0
                
                for i, bit in enumerate(bin_str):
                    # Greedy: If bit is '1', we want '0'. If '0', we want '1'.
                    wanted = '1' if bit == '0' else '0'
                    
                    if wanted in node.children:
                        # Success! We found opposite bit.
                        curr_xor += (1 << (31 - i)) # Add 2^pos
                        node = node.children[wanted]
                    else:
                        # Failed. Must go same path (No addition to XOR).
                        node = node.children[bit]
                        
                max_xor = max(max_xor, curr_xor)
                
            return max_xor
