// Directed Acyclic Graph · DFS vs BFS · When to use what

DAG kya hai?

Directed — edges ek direction mein jaate hain (A→B, B nahi ja sakta A).

Acyclic — koi cycle nahi hoga. Ek baar visit kiya node, wapas nahi aayenge.

Graph — nodes aur edges ka collection.

Real life: Task scheduling, Git commits, build systems (webpack), course prerequisites.


DIRECTED NO CYCLES CYCLE = NOT DAG
DFS — Depth First

Ek path pe jitna deep ja sake, phir backtrack karo.

Stack use karta hai (ya recursion call stack).

Jaise maze solve karna — ek direction mein jaao, dead end mila toh wapas aao.

Best for: Topological sort, cycle detection, pathfinding.

BFS — Breadth First

Pehle saare neighbors visit karo, phir unke neighbors.

Queue use karta hai (FIFO).

Jaise stone ko pond mein daalao — ripples layer by layer failti hain.

Best for: Shortest path, level order, dependency resolution.

🔴 DFS — Step by Step

  • Push start node onto Stack
  • Pop top node, mark visited
  • Push unvisited neighbors onto stack
  • Repeat until stack empty
  • Time: O(V + E) · Space: O(V)

🔵 BFS — Step by Step

  • Enqueue start node into Queue
  • Dequeue front node, mark visited
  • Enqueue unvisited neighbors
  • Repeat until queue empty
  • Time: O(V + E) · Space: O(V)

// Real Interview Scenarios — DFS vs BFS choose karo

DFS
Topological Sort
Task dependencies, build order, course prerequisites. DFS + finish time = topo order.
DFS
Cycle Detection in DAG
Visited + InStack arrays. Back edge milaa toh cycle hai. DFS naturally captures this.
BFS
Shortest Path (unweighted)
Word Ladder, 0-1 BFS, minimum steps. BFS guarantees shortest path in unweighted graphs.
BFS (Kahn's)
Topo Sort alternate
In-degree based. Better when you need level-wise processing or parallel task simulation.
DFS
Connected Components
Islands, regions. DFS faster to implement for simple reachability problems.
BFS
Level Order / Distance
Tree levels, distance from source, multi-source BFS (01-matrix, rotting oranges).
// The Golden Rule, bhai —
Shortest path chahiye? → BFS. Unweighted = BFS. Weighted = Dijkstra.
Dependency order chahiye? → DFS (post-order) ya Kahn's BFS.
Cycle detect karna hai? → DFS with visited+inStack.
Memory concern hai? → DFS (stack ~ depth), BFS (queue ~ width). Wide graph = DFS better. Deep tree = BFS better.
DAG pe DP karna hai? → Topo sort first, phir DP in topo order.
