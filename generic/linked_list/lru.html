Core Concepts
🔗
Why Doubly Linked List?
Deletion from any position in O(1) — dono taraf ke pointers hain (prev + next), toh node ko directly unlink kar sakte hain bina traverse kiye. Singly linked list mein prev nahi hota, O(n) lagta.
🗺️
Why HashMap?
HashMap stores key → Node reference. Directly us node tak pahuncho in O(1). Bina map ke list traverse karna padta — O(n). Dono milao → O(1) everywhere.
📍
Sentinel Nodes (HEAD/TAIL)
Dummy HEAD aur TAIL nodes hain — actual data nahi, sirf boundary markers. Isse edge cases (empty list, single node) automatically handle hote hain. Real nodes hamesha HEAD.next se TAIL.prev ke beech rehte hain.
⚡
MRU = HEAD.next, LRU = TAIL.prev
Most Recently Used hamesha HEAD ke baad insert hota. Least Recently Used hamesha TAIL ke pehle hota. Eviction? Bas tail.prev delete karo — O(1).
🔄
GET = Delete + Re-Add
Jab koi node access hoti hai, woh list se unlink hoti hai aur front pe relink — object wahi ka wahi, sirf pointers change hote hain. HashMap reference unchanged rehta, object alive rehta.
🗑️
Eviction Strategy
Jab capacity exceed ho — tail.prev node evict hoti hai (LRU). Usse DLL se unlink karo, HashMap se delete karo. Node mein key store honi chahiye taki dict se delete kar sako.

Annotated Code
_add & _delete O(1) 
def _add(self, node):
    # Always add after HEAD (MRU pos)
    node.prev = self.head
    node.next = self.head.next
    self.head.next.prev = node  # old front's prev
    self.head.next = node       # head points to new

def _delete(self, node):
    # Wire prev and next around node
    _prev = node.prev
    _next = node.next
    _prev.next = _next   # Bug fix: not `next`
    _next.prev = _prev
get O(1) 
def get(self, key):
    if key in self.cache:
        node = self.cache[key]
        # Move to front (now MRU)
        self._delete(node)  # unlink
        self._add(node)     # relink at front
        # Same object, just rewired!
        return node.value  # .value, not .val
    return -1
put O(1) 
def put(self, key, value):
    if key in self.cache:
        self._delete(self.cache[key])
    new_node = Node(key, value)  # store key!
    self._add(new_node)          # same node add karo
    self.cache[key] = new_node
    if len(self.cache) > self.capacity:
        lru = self.tail.prev
        self._delete(lru)
        del self.cache[lru.key]  # needs key attr
