Multi-Source BFS ka concept: Ek baar mein saare rotten oranges ko queue mein daalo. Phir BFS chalaao — har minute ek "level" hai. Level-by-level spread hota hai rot, exactly like ripples in a pond from multiple stones simultaneously.

Code + Time Complexity — Har line ka hisaab 
def orangesRotting(grid):
    rows, cols = len(grid), len(grid[0])  ← O(1)
    queue = deque()
    fresh = 0

    # ① Grid scan — saare cells ek baar dekho
    for r in range(rows):                 ←
        for c in range(cols):             ←  O(M×N)  — M=rows, N=cols
            if grid[r][c] == 2:           ←  har cell ek baar check
                queue.append((r, c, 0))    ← rotten → queue mein (multi-source!)
            elif grid[r][c] == 1:
                fresh += 1               ← fresh count track karo

    # ② BFS — level by level (minute by minute)
    minutes = 0
    dirs = [(-1,0),(1,0),(0,-1),(0,1)]  ← 4 directions

    while queue:
        r, c, minute = queue.popleft()   ← O(1) per dequeue
        for dr, dc in dirs:             ← 4 neighbors check karo
            nr, nc = r+dr, c+dc
            if (0 <= nr < rows and       ← bounds check O(1)
                0 <= nc < cols and
                grid[nr][nc] == 1):      ← fresh hai?
                grid[nr][nc] = 2         ← rot karo (visited mark)
                fresh -= 1
                minutes = minute + 1
                queue.append((nr, nc, minute+1))  ← next minute
    #  Total BFS: har cell max ek baar queue mein → O(M×N)

    return minutes if fresh == 0 else -1  ← O(1)

Interview Follow-ups — Ye zaroor poochhe jaate hain
Q: Multi-source kyun? Single source kyun nahi?
Agar ek ek rotten orange se separate BFS chalate, toh har source ke liye O(M×N) lagta — total O(R × M×N) jahan R = rotten count. Multi-source BFS sab sources ek saath process karta hai — sirf O(M×N). Real time simulate karta hai — minute 0 pe saare rotten simultaneously spread karte hain.
Q: Visited array kyun nahi banaya?
Grid khud hi visited track kar raha hai! Cell 1 hai = fresh (unvisited), Cell 2 hai = rotten (visited). In-place modification se extra O(M×N) space bachta hai. Agar original grid preserve karna ho toh alag visited array banao.
Q: -1 kab return karte hain?
BFS complete hone ke baad agar fresh > 0 hai toh koi fresh orange reachable nahi tha (empty cell se surrounded tha). Return -1. Agar fresh == 0 already initially, return 0 — no time needed.
Q: Minutes kaise track karte hain?
Do approaches: (1) Queue mein (r, c, minute) store karo — simple. (2) Level-by-level BFS — queue ka current size leke ek poora level process karo, phir minute++. Dono O(M×N) hain. Tuple approach zyada readable hai interviews mein.
Q: 4 directions kyun, 8 kyun nahi?
Problem statement mein specifically 4-directionally adjacent likha hai (up, down, left, right). Diagonal movement allowed nahi. Agar 8-directional hota toh dirs array mein 4 aur tuples add karo.
Q: Course Schedule se analogy?
Bilkul same Kahn's BFS! Rotten orange = node with indegree 0 (no dependency). Fresh neighbors = dependent nodes. Jaise Kahn's mein count track kiya, yahan fresh count track karte hain. Both detect "unreachable" nodes.

"We do a single grid scan O(M×N) to initialize. Then BFS processes each cell at most once — O(M×N). Each cell checks 4 neighbors — still O(M×N) total. Queue holds at most M×N cells. Hence O(M×N) time and space."
