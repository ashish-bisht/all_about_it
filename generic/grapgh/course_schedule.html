STEP 2 — Time Complexity — Har line ka hisaab 
def canFinish(numCourses, prerequisites):
    # ① Adjacency list + indegree initialize
    adj = defaultdict(list)          ← O(V)  — V nodes ke liye
    indegree = [0] * numCourses       ← O(V)  — V size array

    # ② Edges process karo
    for course, pre in prerequisites:  ← O(E)  — E = len(prerequisites)
        adj[pre].append(course)       ← O(1)  amortized per append
        indegree[course] += 1         ← O(1)  per operation
    #                         Total so far: O(V + E)

    # ③ Queue mein 0-indegree nodes daalo
    queue = deque([i for i in range(numCourses)
                   if indegree[i] == 0])   ← O(V)

    count = 0
    # ④ BFS loop
    while queue:
        node = queue.popleft()       ← O(1) — har node ek baar hi aayega
        count += 1                   ← O(1)
        for neighbor in adj[node]:  ← Total across ALL iterations = O(E)
            indegree[neighbor] -= 1  ← O(1)
            if indegree[neighbor] == 0:
                queue.append(neighbor)← O(1)
    #                         Total BFS: O(V + E)

    return count == numCourses     ← O(1)

STEP 3 — Interview mein poochhe jaane wale follow-ups
Q: Har edge ek baar kyun?
Jab hum node ko process karte hain, uski adj list ek baar traverse hoti hai. Node ko queue se nikalne ke baad woh dobara queue mein nahi aata (indegree sirf decrease hoti hai, 0 se neecha nahi jaati). Toh total edge traversals = E.
Q: Space O(V) ya O(V+E)?
O(V+E) — adj list mein total E entries hain (ek per directed edge). V nodes + E edge entries = V+E. Agar dense graph ho (E ≈ V²), toh space O(V²) ho sakta hai practically.
Q: Cycle kaise detect hoti hai?
Cycle mein phansa node ka indegree kabhi 0 nahi hoga (uska predecessor bhi cycle mein hai, uska bhi process nahi hoga). Toh count < numCourses → cycle hai.
Q: Course Schedule II kaise?
Same code, bas result = [] banao aur har node process hone pe result.append(node). End mein return result if len==n else []. Result automatically topological order mein hoga!
Q: V aur E worst case kya?
V = numCourses (up to 2000 in LC). E = prerequisites.length (up to 5000 in LC). Theoretically E can be V*(V-1) in dense directed graph. Isliye O(V+E) likhte hain, O(V²) nahi — sparse graph assume karte hain.
Q: Adjacency Matrix kyun nahi?
Matrix space = O(V²). Dense graphs ke liye okay, but sparse graphs pe wasteful. Edge traversal bhi O(V) per node ho jaata (pura row scan karo). Adj list preferred for sparse graphs.
