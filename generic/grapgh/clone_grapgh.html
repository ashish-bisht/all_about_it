Clone Graph

// HashMap + BFS/DFS · Deep Copy · O(V+E) Time & Space

Ye problem mein actually kya karna hai?
Problem
Ek undirected graph diya hai as adjacency list. Har node ke paas val aur neighbors[] hai.

Deep copy banao — naye nodes, naye neighbor pointers. Original graph ka koi reference nahi hona chahiye clone mein.
The Catch — Cycle!
Graph mein cycles ho sakti hain. Agar sirf recursion karo bina tracking ke → infinite loop!

Solution: HashMap rakhte hain jisme original_node → clone_node mapping hoti hai. Agar node already clone ho chuki hai, wahi return karo.
HashMap = Visited + Memo
HashMap double kaam karta hai:
1. Visited — ye node dekh li hai
2. Memoization — clone node already bani hai, reuse karo

Rotten Oranges mein grid khud visited tha, yahan HashMap visited hai.

def cloneGraph(node):
    if not node: return None          ← edge case

    cloned = {}                         ← HashMap: original → clone
    cloned[node] = Node(node.val)       ← ① pehla node clone karo
    queue = deque([node])               ← ② BFS start karo original se

    while queue:
        curr = queue.popleft()          ← original node

        for neighbor in curr.neighbors:
            if neighbor not in cloned:
                cloned[neighbor] = Node(neighbor.val)  ← naya clone banao
                queue.append(neighbor)  ← BFS mein add karo

            # clone ka neighbor = cloned version of original neighbor
            cloned[curr].neighbors.append(cloned[neighbor])

    return cloned[node]               ← starting node ka clone return karo

# Time: O(V+E) — har node ek baar, har edge ek baar
# Space: O(V) — HashMap mein V entries, Queue max V nodes
def cloneGraph(node):
    cloned = {}                         ← HashMap: original → clone

    def dfs(node):
        if not node: return None

        if node in cloned:              ← ① already cloned? return it!
            return cloned[node]           (yahi cycle handle karta hai)

        clone = Node(node.val)          ← ② naya node banao
        cloned[node] = clone            ← ③ PEHLE map mein daalo (cycle se pehle!)

        for neighbor in node.neighbors:
            clone.neighbors.append(     ← ④ recursively clone neighbor
                dfs(neighbor)
            )

        return clone

    return dfs(node)

# Key insight: cloned[node] = clone ko DFS jaane SE PEHLE set karo
# Warna cycle mein infinite recursion hoga!
# Time: O(V+E) · Space: O(V) — hashmap + call stack

⏱ Time Complexity

Node visit (clone banana)

O(V)
Neighbor traversal
Har edge ek baar process hoti hai — neighbor loop mein. Undirected hai toh har edge 2 baar dikhti hai but still O(E).
O(E)
TOTAL TIME
O(V + E)

💾 Space Complexity

HashMap (cloned dict)
V nodes ke liye V entries — ek per original node.
O(V)
BFS Queue / DFS Call Stack
BFS: worst case V nodes queue mein. DFS: call stack depth = longest path = O(V) worst case.
O(V)
Cloned nodes + edges
Output graph khud V nodes + E edges store karta hai — but ye output space hai, usually separate count karte hain.
O(V+E)
AUX SPACE
O(V)
HashMap + queue/stack. Output space alag.
Interview Follow-ups
Q: HashMap pehle kyun set karte hain DFS mein?
Cycle handle karne ke liye. Agar node 1 → node 2 → node 1 cycle hai, aur hum 1 ko process kar rahe hain uske neighbor 2 mein jaate hain, 2 se wapas 1 pe aate hain. Agar HashMap mein 1 already hai → return karo. Nahi hai → infinite loop. Isliye cloned[node] = clone neighbors se PEHLE.
Q: BFS vs DFS — kaunsa better?
Dono O(V+E). BFS slightly safer in interviews — no stack overflow risk for large graphs. DFS elegant hai but very deep graph (V=10^5) pe Python mein recursion limit hit ho sakta hai. sys.setrecursionlimit ya iterative DFS use karo.
Q: Shallow copy vs Deep copy?
Shallow copy: Same node objects, bas pointers copy kiye — original graph hi return ho jaata.
Deep copy (ye problem): Brand new Node objects banate hain same values ke saath, naye neighbor pointers. Original aur clone completely independent.
Q: Rotten Oranges se kya alag hai?
Rotten Oranges mein grid khud visited tracker tha (cell value 2 = visited). Clone Graph mein nodes immutable hain — unhe modify nahi kar sakte. Isliye external HashMap chahiye. Plus HashMap double kaam karta hai — visited + actual clone store.
Q: null/None node handle kaise?
Edge case — if not node: return None. Empty graph possible hai. Ye pehli cheez check karo. BFS mein starting node ke saath hi queue mein daal dete hain, toh null already handle ho jaata hai.
Q: Node class kaise hoti hai?
class Node:
  def __init__(self, val):
    self.val = val
    self.neighbors = []
Undirected graph hai — agar edge 1-2 hai toh 1.neighbors mein 2 bhi hai aur 2.neighbors mein 1 bhi.
